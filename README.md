1. 가변형 멀티 프리팹 재사용 스크롤 (RecycleScrollCore.cs)
   
방대한 리스트(가챠 결과, 대규모 인벤토리, 채팅 등)를 스크롤할 때 발생하는 프레임 드랍과 GC 스파이크를 해결하기 위해 설계된 고도화된 스크롤 뷰 컴포넌트입니다.

stackalloc 활용 최적화: 화면이 갱신될 때마다 발생하는 잦은 큐(Queue) 연산에서 배열이나 List 대신 stackalloc을 사용하여 스택 메모리에 인덱스를 할당했습니다.
이를 통해 스크롤 시 발생하는 힙 메모리 할당을 완벽하게 제거하여 미세한 프레임 끊김 현상을 방지했습니다.

가변 사이즈 동적 풀링: 모든 아이템의 크기가 동일하다는 가정을 버리고, 런타임에 동적으로 크기를 계산(GetItemSize())하여 정확한 위치에 객체를 재배치합니다.
여러 종류의 프리팹을 동시에 섞어서 풀링할 수 있는 확장성을 제공합니다.


2. 비동기 캐릭터 리소스 풀 매니저
   
아웃게임(로비, 락커룸 등)에서 빈번하게 일어나는 3D 캐릭터 및 장착 파츠(스킨) 교체 시의 메모리 누수와 비동기 레이스 컨디션을 방지하는 스마트 풀링 매니저입니다.

변경된 파츠만 선별하는 스마트 풀링 (최적화):
캐릭터 교체 시 객체를 통째로 파괴/생성하지 않습니다. 기존 착용 파츠와 비교하여 '변경된 파츠만' 선택적으로 Addressables 비동기 로드를 수행하고, 
중복되는 파츠는 Renderer만 켜고 끄는 방식으로 렌더링 부하와 로딩 속도를 극적으로 개선했습니다.

파편화된 로직 중앙화(Centralize)를 통한 협업 생산성 증대 (가독성/안정성):
코드 이곳 저곳에 산재되어 파편화된 캐릭터 생성 로직을 Spawn()이라는 단일 파이프라인으로 완벽하게 통합했습니다.
이를 통해 팀원들은 내부의 복잡한 비동기 처리나 큐(Queue) 제어를 전혀 신경 쓸 필요 없이 Spawn() 함수 하나만 호출하여 안전하게 캐릭터를 띄울 수 있게 되었고,
팀 전체의 작업 효율과 코드 유지보수성이 크게 향상되었습니다.
